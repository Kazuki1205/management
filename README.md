# Management(生産管理アプリ)について

　工場の生産管理業務を、一通り処理可能なアプリケーションです。  


 - 業務に必要な情報を管理する各種マスタ項目(社員・部署・商品・顧客)を実装。
 
 
 - 商品を新たに製作開始する製作手配項目を実装。
 
 
 - 各現場社員が製作した記録を入力する日報入力項目を実装。
 
 
 - 加工が完了した商品情報を入力する入庫入力項目を実装。
 
 
 - 営業社員が請けた注文情報を入力する受注入力項目を実装。
 
 
 - 出荷可能な受注明細に対して出荷処理をする出荷手配項目を実装。
 
 
 - 製作中の製番毎の進捗状況を表示する進捗一覧項目を実装。
 
 
 - 商品毎の実在庫・注残数・有効数を表示する在庫一覧項目を実装。


# 背景

　前職では工場の生産管理の仕事をしていました。  
主な職務内容は、製作手配～在庫管理に至るまで、それらの作業を主にExcelを用いて行っており、作業毎にファイルが作られていた為、全体を通しての管理が難しい状態でした。  
そういった経験から製作手配～在庫管理まで統合した、一通りの管理が出来るアプリケーションを作成してみようと思いました。

# 各種機能動作


全般的に削除は論理削除とし、行が灰色に変化する。


 - 社員マスタ
    - ADMIN権限のみ、閲覧・登録・更新・削除可能。
    - 社員情報を管理する。
    - 社員IDは「0001」の形式で自動採番される。
    - 社員ID「0001」のみADMIN権限を持たせている為、編集不可。
    - 登録時の部署に応じて付与する権限を変える。
       - 「間接」→ROLE_OFFICE
       - 「旋盤,切削,焼入,研磨,検査,組立,梱包」→ROLE_FIELD
       - 「その他部署」→ROLE_SALE  
         
         
 - 部署マスタ
    - ADMIN権限のみ、閲覧・登録・更新・削除可能。
    - 部署情報を管理する。
    - 部署コードは「0001」の形式で自動採番される。
    - 部署と1:1の関係にある社員情報を取得し、所属人数・所属社員を確認可能。
    
    
 - 商品マスタ
    - 全ての権限で閲覧可能。ADMIN・OFFICE権限のみ登録・更新・削除可能。
    - 商品情報を管理する。
    - 商品コードは「0000001」の形式で自動採番される。


 - 顧客マスタ
    - 全ての権限で閲覧可能。ADMIN・OFFICE権限のみ登録・更新・削除可能。
    - 顧客情報を管理する。
    - 顧客コードは「0001」の形式で自動採番される。
    - 郵便番号フォームに正しく入力すると、都道府県・市区町村が自動取得される。
    
    
 - 製作手配
    - 全ての権限で閲覧可能。ADMIN・OFFICE権限のみ登録・更新・削除可能。
    - 商品・製作数・完成予定日を設定し、登録する。
    - 製作番号は、年4桁-月2桁-当該年月の通し番号4桁(例 2022-01-0001)の形式で自動採番される。
    - 製作番号に紐づいた不良数計・入庫数計を表示する。
    - 「不良数計 + 入庫数計 = 製作数」となったタイミングで製作完了とし、完成日が表示される。
    - 製作完了された行は青色に変化する。
    
    
 - 日報入力
    - 全ての権限で閲覧可能。ADMIN・FIELD権限のみ登録・更新・削除可能。
    - 登録した社員のみが、当該情報を編集可能。
    - 製作中の製品を選び、各部署毎の完了数・不良数を登録する。
    
    
 - 入庫入力
    - 全ての権限で閲覧可能。ADMIN・OFFICE権限のみ登録可能。
    - 製作中の製品を選び、加工が完了した物の入庫数量を登録する。同時に在庫テーブルの実在庫を増加させ、  
    「入庫数計 + 不良数計 = 製作数」となっていれば、製作製番を製作完了とする。


 - 受注入力
    - 全ての権限で閲覧可能。ADMIN・SALE権限のみ登録・更新・削除可能。
    - 登録した社員のみが、当該情報を編集可能。
    - 顧客を選び、受注した商品情報を入力する。商品点数に応じて明細行を追加し、商品を選び受注数を登録する。
    - 「出荷済計 = 受注数」となったタイミングで、明細行の出荷完了とする。全ての明細行が出荷完了または削除されたタイミングで、受注自体を出荷完納とする。
    - 受注番号は「00000001」の形式で自動採番される。
    - 入力画面で単価・受注数を変更すると、小計が計算される。
    - 詳細画面で、当該注文の明細行を確認できる。
    - 出荷完了した行は青色に変化する。
    
    
 - 出荷手配
    - 全ての権限で閲覧可能。ADMIN・OFFICE権限のみ登録可能。
    - 受注番号を選び、受注明細毎に出荷数を登録する。同時に在庫テーブルの実在庫を減少させ、  
    「出荷数計 = 受注数」となったタイミングで、明細行の出荷完了とする。全ての明細行が出荷完了または削除となっていれば、受注自体を出荷完納とする。  
    実在庫以上の出荷数は登録不可。
    - 入力画面で出荷数を変更すると、出荷金額が計算される。
    - 出荷済の明細行は検索時表示されない。
    
    
 - 進捗一覧
    - 全ての権限で閲覧可能。
    - 製作中の製作製番のみを一覧表示する。日報履歴から情報を取得し、日報入力があれば「完了数、不良数、更新日」を表示する。  
    進捗状況に応じて行が青色に変化する。製作完了となると一覧に表示されなくなる。


 - 在庫一覧
    - 全ての権限で閲覧可能
    - 商品毎の実在庫・注残数・有効数を表示する。
    - 注残数は、当該商品に紐づいた受注明細情報から、未出荷数を集計した物を表示。
    - 有効数は、当該商品の「実在庫 - 注残数」を計算した、有効な在庫数を表示。
    - 詳細画面で、当該商品の注残状況を確認できる。
